Detailed Description

This API allows user to use multiple instances of libtrap in the same process.

The API is similar to Simple API. The difference is in the private context memory that is returned by trap_ctx_init() and is freed by trap_ctx_finalize(). Obtained context pointer must be passed to all functions from context API.

Read data from input interfaces according to ifc_mask.

Read a record from one of interfaces specified by ifc_mask and store pointer to it into data. If data are not available on any of specified interfaces, wait until data are available or timeout elapses. If timeout < 0, wait indefinitely. When function returns due to timeout, contents of data and size are undefined.

Parameters
[in]ctxPointer to the private libtrap context data (trap_ctx_init()).
[in]ifc_maskMask of interfaces to listen on (if i-th bit is set, interface i is enabled).
[out]dataPointer to received data. The result is an array of trap_multi_result_t. The size of array is equal to the number of input IFCs.
[out]sizeSize of data in bytes containing the size of trap_multi_result_t array in bytes.
Returns
Error code - 0 on success, TRAP_E_TIMEOUT if timeout elapses.
Note
Data must not be freed! Library stores incomming data into static array and rewrites it during every trap_get_data() call.
See also
trap_ctx_ifcctl, trap_multi_result_t

Send data via output interface.

Write data of size size given by data pointer into interface ifc. If data cannot be written immediately (e.g. because of full buffer or lost connection), wait until write is possible or timeout microseconds elapses. If timeout < 0, wait indefinitely.
